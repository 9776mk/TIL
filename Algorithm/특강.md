Q & A
Q. 안 풀리는 문제 얼마나 고민?
- 30분 ~ 1시간, 단계별로 다른 모습이 있음. 무엇이 힘든지 보세요

Q. 문제가 이해 안된다. 그때 어떻게?
- 제약 조건 만들어야 함(논리)
- input => 손으로 실행 => 결과

Q 고민 vs 구글링
- 커뮤니티(디스코드)

- 엣지 케이스를 물어본다.
- 구글링 -> 내가 어떤 논리를 놓쳤느냐?

Q 하나의 문제 vs 다음 단계
- 다음 단계

Q. 디버깅
- 내가 활용하는 변수
  - 원래 안되었어요? 지금 안 된거에요?
  - 지금 안된거면 뭐하다가 그랬어요?
  - 혹시 그럼 다시 한번 그거 해보시곘어요? 똑같나요?
  - 한번 껐다 키세요. => 반복적으로 발생하는지
  - 그거 저 줘보세요 => 저사람 컴퓨터 문제인지 아니면 공통 문제인지
  - 뭐 설치하셨어요? 이전에 뭐 하셨어요?

Q. 중간 개념이나 힌트를 보고 푼 문제 나중에 다시 풀어야 하나요?
